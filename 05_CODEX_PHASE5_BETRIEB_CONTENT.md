# CODEX PHASE 5 – Betrieb, Content und Wartung

## Ziel

SmashTime soll langfristig ohne Codeänderungen gepflegt werden können.

## Aufgaben

### Admin-Dokumentation
- Wie legt man ein Event an?
- Wie legt man News an?
- Wie pflegt man Fightcards?
- Wie lädt man Bilder hoch?
- Wie archiviert man Events?

### Content-Regeln
- keine fremden Fighterbilder ohne Freigabe
- Champions nur in Champion-Kontext
- Eventdaten zentral pflegen
- Ticketpreise nur offiziell
- Sponsorlogos nur mit Freigabe

### Betrieb
- Backup-Konzept
- Rollen/Rechte
- monatliche QA
- Bildgrößen-Regeln
- Deployment-Prozess
- Änderungsprotokoll

### Event-Workflow
1. Event anlegen
2. Ticketseite aktivieren
3. Fightcard vorbereiten
4. News veröffentlichen
5. Event live halten
6. nach Event Ergebnisse und Galerie ergänzen
7. Event archivieren

## Fertig, wenn

- neue Events ohne Code erstellt werden können
- News ohne Code veröffentlicht werden können
- Fightcards ohne Code gepflegt werden können
- Sponsoren ohne Code gepflegt werden können
- Kontaktanfragen sauber verarbeitet werden können
